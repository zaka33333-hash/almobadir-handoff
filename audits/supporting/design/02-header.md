# Header (.hdr3) — Forensic Audit

**Audited:** 2026-04-26
**Scope:** announce strip, main bar, mega menu (network), search, lang toggle, CTA, hamburger, mobile drawer, sticky/scroll behaviour
**Skills consulted:** design:design-critique, typography (Arabic+Latin), web-design-guidelines, design-audit
**Inputs:** `website/index.html` lines 35–136, `website/assets/v3.css` lines 9–117, `website/assets/v3.js` lines 10–70, `website/assets/tokens.css`, plus live DOM measurements at 1440 / 880 / 390 and `design-audit/screenshots/hdr3-{1440,768,390}.png`.
**Benchmarks referenced:** Stripe.com nav, Linear.app nav, Apple.com top bar, NYT masthead, Mailchimp, Vercel, Notion.

---

## Executive Summary

The header is **architecturally ambitious** — sticky-with-blur, three-state (announce + main + scrolled), a mega menu, a slot for a ⌘K search, an animated ticker, and a colorful drawer for mobile. The visual rhythm at 1440 reads like a Stripe-class top bar: 72-px tall, 24-px gutters, a 1320-px container, the wordmark anchored on the right (correct for RTL), nav centered, utility cluster on the left. The **crimson CTA pulls properly as a hot stop**; the lang pill is a quietly elegant little component; the announce strip is a real piece of "marketing surface" that most premium sites still don't bother with. There is taste here.

The **execution gap is large**. The mega menu is **structurally broken**: the `.hdr3-mega` element sits OUTSIDE the `<li class="hdr3-has-mega">` it should be a descendant of (HTML lines 67–74 close the `<li>` then expose `.hdr3-mega` as a sibling), so the CSS rule `.hdr3-has-mega.is-open .hdr3-mega { opacity: 1 }` (v3.css:54) **never matches** and the network mega never opens — verified in DOM. The mobile drawer's `top: 60px` (v3.css:98) is wrong by 44 px when the page isn't scrolled (header is 104 px tall, not 60), so the drawer slides under a still-visible header. **Every interactive in the header has its focus outline killed** (`outline-style: none` computed on every focusable, no `:focus-visible` ring defined) — a categorical accessibility miss. Touch targets are 40×40 (burger) and ~36×30 (CTA at ≤880 px) — both below the 44×44 minimum. Plus token literals (`100` for z-index instead of `--z-header`), unsynced search-shortcut hint (`⌘ K` shown to all OS users including Windows/Arabic-keyboard speakers), and the search input is a decorative box with no submit, no suggestions, and no aria-keyshortcut.

What's blocking marketing: **the network mega menu is the highest-conversion artefact in the header** — six branded IG accounts with stat hierarchies; it is the proof of "1.54M reach" the site sells. It does not open. Every visitor who hovers الشبكة gets nothing. That alone is a P0. Combined with the broken drawer geometry on mobile (where ≥70% of GCC traffic lives), the navigation is non-functional on both ends of the spectrum.

## Token compliance score: 4/10

The header references `--ink`, `--crimson`, `--ink-soft`, `--ink-mute`, `--hairline`, `--ease-glide`, `--font-ar-display`, `--font-ar-body`, `--font-mono` — that's the right list.
